## 2.1 总体研究框架

```
文献收集（35篇PDF）
    ↓
文本提取（pypdf）
    ↓
文本分块（chunk_size=500, overlap=50）
    ↓
向量化（sentence-transformers / paraphrase-multilingual-MiniLM-L12-v2）
    ↓
向量存储（numpy 数组）
    ↓
语义检索（余弦相似度 / DeepSeek 本地模型）
    ↓
知识问答 + 信息图表可视化
```

## 2.3 本地化大模型技术

### 2.3.1 为什么选择本地化部署

| 方案 | 优势 | 劣势 |
|------|------|------|
| **本地部署（我们的选择）** | 隐私保护、无网络依赖、可离线运行 | 需要本地算力 |
| OpenAI API | 能力强 | 需要联网、有隐私风险 |
| 国内商业API | 中文能力强 | 依赖第三方服务 |

### 2.3.2 模型选型

**向量模型**：`paraphrase-multilingual-MiniLM-L12-v2`

- 由 sentence-transformers 提供
- 支持50+种语言（包括中文）
- 384维向量输出，速度快
- 模型大小约500MB，适合消费级GPU

**语言模型**（可选）：DeepSeek-Chat 系列

- 支持中文上下文理解
- 可本地部署（推荐 DeepSeek-Coder-Instruct）

### 2.3.3 RAG 检索增强流程

```
用户提问
    ↓
将问题向量化（MiniLM）
    ↓
在知识库向量中检索 Top-K 最相关内容
    ↓
将相关内容作为上下文注入 Prompt
    ↓
语言模型生成回答
    ↓
返回带来源标注的回答
```

## 2.4 数据处理流程

### 2.4.1 PDF 文本提取

```python
# 使用 pypdf 提取文本
with open(pdf_path, 'rb') as file:
    pdf_reader = pypdf.PdfReader(file)
    text = ""
    for page in pdf_reader.pages:
        text += page.extract_text() + "\n"
```

### 2.4.2 文本分块策略

- **chunk_size**：500词/块
- **overlap**：50词重叠（保证跨块语义连贯）
- **去噪处理**：移除页眉页脚、空行、注释

### 2.4.3 向量化与存储

```python
# 批量向量化
from sentence_transformers import SentenceTransformer

model = SentenceTransformer('paraphrase-multilingual-MiniLM-L12-v2')
embeddings = model.encode(documents)  # shape: (n_chunks, 384)
```

## 2.5 信息图表设计方法

### 2.5.1 设计原则

- **数据驱动**：所有可视化数据来自实际文献内容
- **文化尊重**：色彩方案参考敦煌矿物颜料体系
- **交互友好**：支持筛选、悬停提示、点击详情

### 2.5.2 主要图表类型

| 图表 | 工具 | 数据来源 |
|------|------|---------|
| 历史时间线 | HTML + CSS + JS | 文献梳理 |
| 雷达图（纹样频率） | Chart.js | 35篇文献主题统计 |
| 饼图（文献分布） | Chart.js | 35篇文献主题分类 |
| 柱状+折线图（洞窟研究） | Chart.js | 文献引用数据 |
| 色彩分布折线图 | Chart.js | 色彩配置研究文献 |

## 2.6 研究局限性

1. **文献覆盖不全**：35篇文献无法覆盖所有敦煌藻井研究成果
2. **向量化精度**：MiniLM 模型为通用多语言模型，非敦煌学专业模型
3. **图像数据缺失**：暂未引入藻井图片的视觉分析
4. **实时性限制**：知识库更新需要重新运行脚本
